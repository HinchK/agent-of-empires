Generated by /plan-eng-review on 2026-04-10

## Affected Pages/Routes
- `/api/*` (all REST endpoints) — verify remote access middleware blocks non-local when toggle off
- `/sessions/{id}/ws` (WebSocket) — verify remote WebSocket connections blocked when toggle off
- Desktop webview `http://127.0.0.1:<port>/?token=<token>` — verify loads after server ready signal
- QR popover window — verify dark mode, arrow positioning, network change updates
- macOS notifications — verify fires on state transition, debounce, permission handling

## Key Interactions to Verify
- Remote access toggle: off→on enables LAN connections, on→off blocks instantly
- QR code popover: appears on toggle-on with arrow pointing at tray, disappears on toggle-off
- AirDrop share: share sheet opens (or Copy URL fallback if unavailable)
- Notification: appears within 4s of agent entering waiting state, click focuses window
- Tray tooltip: updates within 2s with session count + status
- Network change: QR regenerates within 10s, notification warns phone users
- CLI coexistence: `aoe serve` detects running desktop server and defers

## Edge Cases
- Port 8080 occupied → app finds next available (8081-8090)
- Multiple LAN interfaces (WiFi + VPN + Docker) → correct IP in QR
- Server crash mid-session → tray grays out, restart button works
- No network → remote toggle disabled, "no network" message
- Notification permission denied → silent degrade, no crash
- Network IP unchanged on poll → no spurious QR regeneration
- Zero sessions → "No sessions" tooltip, gray icon
- Broadcast channel with no subscribers → no panic
- Ready signal fails (bind error) → Tauri shows error dialog

## Critical Paths
- Download .dmg → drag to Applications → double-click → see dashboard in <30s
- Toggle remote → scan QR on phone → dashboard opens with auth cookie set
- Agent waits for input → notification appears → click opens dashboard to that session
- Network changes → QR auto-updates → user re-scans → new connection works
