## Vision

### 10x Check
The 10x version is a full "AI agent control plane": download a .app, it manages all your agents across all your machines, accessible from any device, with built-in secure tunneling, push notifications, and a team dashboard. One app to rule them all.

The current plan is the foundation: native macOS app + QR pairing. The 5 accepted expansions push it from "wrapper" to "feels like a real product."

## Expansion Implementation Details

### #1: Dark mode for QR popover (S)
**Approach:** The QR popover is a frameless Tauri window rendering HTML. Add `@media (prefers-color-scheme: dark)` CSS rules: dark background (#1e1e1e), light text, inverted QR code border.
**Dependencies:** None new.
**Failure modes:** None (CSS-only, degrades to light mode).
**Success criterion:** Popover matches macOS system appearance.

### #2: Dynamic tray icon + tooltip (S)
**Approach:** On each status poll cycle (every 2s), call `tray_icon.set_tooltip()` with formatted count string and `tray_icon.set_icon()` to swap between pre-rendered icon variants.
**Icon semantics (reconciled with base plan):** green = all sessions running, yellow = any session waiting for input, red = any session errored, gray = server stopped/crashed. Remote access state is shown as a separate menu item text, not icon color.
**Dependencies:** 3 PNG icon variants (green/yellow/red/gray) in `desktop/icons/tray/`.
**Failure modes:** Poll returns empty instances: show "No sessions" tooltip, gray icon.
**Success criterion:** Tray tooltip updates within 2s of session state change.

### #3: Agent attention notifications (M)
**Approach:** Use `tauri-plugin-notification` (maintained, Tauri 2.0 compatible). In the poll loop, track previous session states. When any session transitions from non-waiting to "Waiting" (detected via pane content or status field), fire a notification: title "Agent of Empires", body "{agent_name} is waiting for input in {project_name}". Click opens/focuses the main window.
**Dependencies:** `tauri-plugin-notification = "2"` in desktop/Cargo.toml. Requires `notification` capability in `desktop/capabilities/default.json`.
**Failure modes:** User denied notification permission: degrade silently, log warning. Notification spam: debounce max 1 per session per 60s.
**Success criterion:** Notification appears within 4s of agent entering waiting state.

### #4: Share via AirDrop (M, SPIKE REQUIRED)
**Approach:** Tauri does not expose NSSharingServicePicker natively. Two options: (a) Write a custom Tauri plugin with Swift/ObjC FFI, or (b) use `osascript` to invoke the share sheet as a workaround. Option (b) is fragile. A spike is needed to determine feasibility before committing.
**Fallback:** If the spike fails, replace with a prominent "Copy URL" button (delivers 80% of the value).
**Dependencies:** Potentially a custom Tauri Swift plugin.
**Failure modes:** AirDrop unavailable (older Macs, Bluetooth off): hide the Share button, show only QR + Copy URL. Share sheet invocation fails: fall back to Copy URL.
**Success criterion:** URL can be shared to an iPhone via AirDrop from the QR popover.

### #5: Network change detection (M)
**Approach:** Polling, not callbacks. A tokio task checks `local_ip_address::local_ip()` every 5 seconds, compares against the stored LAN IP in a shared `Arc<RwLock<IpAddr>>`. On change: update the stored IP, regenerate QR code, update the popover if open, fire a notification: "Network changed. QR code updated."
**Why polling over callbacks:** SCNetworkReachability requires CFRunLoop integration which conflicts with tokio. Polling every 5s is simple and the latency is acceptable (QR goes stale for at most 5s).
**Interaction with existing sessions:** When the IP changes, existing phone sessions break (they're connected to the old IP). The notification text includes: "Devices connected to the previous address will need to re-scan." The auth token stays the same.
**Dependencies:** No new deps (already using `local-ip-address`).
**Failure modes:** All interfaces disappear (airplane mode): disable remote toggle, show "No network" in tray.
**Success criterion:** QR code URL updates within 10s of network change.
